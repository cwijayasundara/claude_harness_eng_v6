What it is

Attention & policy — not model failure

A strong coding agent is a powerful hammer. Without explicit norms, everything becomes a nail—including work that is faster in a human UI or unsafe without SME truth.

Thesis: productivity is routing, not “use the agent more.”

What agents are (and aren’t) for

For
  • Stories against written acceptance criteria
  • Tests, fixtures, pure-structure refactors
  • Reproduce → isolate → fix under repo tools
  • Brownfield maps, mechanical migrations + gates
Not by default
  • System of record for ADO / Jira / Linear
  • Inventing regulatory or fund-document truth
  • Self-approving production readiness
  • Dashboard cosplay (UI you can open in 3s)

UI navigation you already have → human. Synthesis across many files under a contract → agent.
