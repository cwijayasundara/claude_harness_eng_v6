# LinkedIn publishing kit — Productive Agentic Engineering

Assets to post the white paper without dumping the full HTML into LinkedIn.

| File | Use |
|------|-----|
| [`agentic-engineering-linkedin-article.md`](./agentic-engineering-linkedin-article.md) | Paste into **LinkedIn → Write article** (~1,500 words) |
| [`agentic-engineering-linkedin-deck.html`](./agentic-engineering-linkedin-deck.html) | Open in Chrome → **Print → Save as PDF** → LinkedIn **document post** |
| [`../agentic-engineering-whitepaper.html`](../agentic-engineering-whitepaper.html) | Canonical full paper (link in comments) |

## Document post (highest reach) — 5 minutes

1. Open `agentic-engineering-linkedin-deck.html` in **Chrome**.
2. **Print** (`⌘P`) → Destination **Save as PDF**.
3. Settings:
   - Layout: **Landscape**
   - Margins: **None**
   - **Background graphics: ON**
   - Scale: **100%** (or Default)
4. Save e.g. `Productive-Agentic-Engineering.pdf`.
5. LinkedIn → **Start a post** → **Add a document** → upload PDF.
6. Title: `Productive Agentic Engineering — without agent psychosis`
7. Caption (short):

```text
Coding agents created an overuse problem, not an underuse problem.

You're not Team Z or Team L. Every task gets a place on the continuum.
Placing it correctly is the skill.

• Domain systems (PE / BFS / insurance) need human-owned BRDs
• Pure tech (doc intelligence) can lean harder on agents
• Sweet spot: human reqs + design → agent generation → band-depth review

Full white paper in comments.
```

8. First comment: links to full HTML + Z/L rubric (from the article file).

## Article post — 10 minutes

1. LinkedIn → **Write article**.
2. Title + body from `agentic-engineering-linkedin-article.md`.
3. Add a simple cover (continuum bar or dark title slide screenshot from the deck).
4. Publish → share as a post.
5. First comment: repo links.
